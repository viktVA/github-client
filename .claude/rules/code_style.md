# Стиль кода — React + TypeScript

## Главное правило
Перед тем как создать что-то новое — проверь не существует ли уже похожего в проекте.
Это касается компонентов, утилит, типов, хуков.

## Структура компонентов
- Один компонент — одна папка: `ComponentName/ComponentName.tsx` + `index.ts` + `ComponentName.module.scss`
- `index.ts` — только реэкспорт, никакой логики
- Типы пропсов — `ComponentNameProps` в том же файле что и компонент
- Компоненты — функциональные, `React.FC<Props>`
- - Компонент который используется только внутри одной страницы — кладётся в src/App/pages/НазваниеPage/components/, а не в src/components/
- В src/components/ только то что используется в 2+ местах

## TypeScript
- Никаких `any` — если не знаешь тип, спроси меня
- Типы для внешних данных (API) — в `src/custom_types/`
- Именование типов: `PascalCase`, суффикс `Info` для расширенных объектов

## Импорты
- Только алиасы: `@components/`, `@api/`, `@utils/`, `@config/`, `@custom_types/`, `@assets/`
- Никаких относительных путей `../../`
- Порядок импортов: внешние библиотеки → алиасы → стили

## Стили
- Только CSS Modules: `styles['block__element']`
- Методология BEM: `block`, `block__element`, `block__element_modifier`
- Несколько классов — через `classNames()` из пакета `classnames`
- Никаких inline-стилей кроме динамических значений (ширина, цвет из переменной)

## Создание компонентов и страниц
- Новый компонент — только через скилл /new-component
- Новая страница — только через скилл /new-page
