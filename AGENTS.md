
# Руководство по репозиторию

Это руководство предназначено для участников, работающих над GitHub Client SPA. Старайтесь вносить изменения локально, воспроизводимо и в соответствии с текущим стеком: React + TypeScript + Vite.

## Структура проекта и организация модулей

* Точка входа: `src/main.tsx` — подключает роутер (`@config/routesConfig`) и глобальные стили.
* Основной layout: `src/App/App.tsx` с header и `<Outlet />`; маршруты определены в `src/config/routes*.tsx`.
* UI-компоненты: `src/components/*` (папки в PascalCase; SCSS-модули для локальных стилей) + общие переменные в `src/components/variables.scss`.
* Страницы: `src/App/pages/RepoListPage` и `src/App/pages/RepoPage` содержат логику экранов.
* API / утилиты: `src/api` — запросы к GitHub, `src/utils` — вспомогательные функции, `src/custom_types` — общие типы TypeScript.
* Стили: глобальные сбросы и токены в `src/styles`; ассеты (иконки) — в `src/assets`.

## Команды сборки, тестирования и разработки

* `npm install` (или `npm ci`) — установка зависимостей.
* `npm run dev` — запуск dev-сервера Vite с HMR.
* `npm run build` — проверка типов (`tsc -b`) и сборка оптимизированной версии в `dist/`.
* `npm run preview` — локальный просмотр собранного приложения.
* `npm run lint` / `npm run lint:fix` — ESLint (flat config) для `*.{ts,tsx}`; используйте перед коммитом.

## Стиль кода и соглашения по именованию

* Язык: TypeScript + функциональные компоненты React; предпочтительно использовать хуки.
* Импорты: используйте алиасы путей (`@components/*`, `@config/*`, `@styles/*` и т.д.), определённые в `tsconfig.json`; избегайте глубоких относительных импортов.
* Компоненты и файлы: PascalCase для компонентов/папок, camelCase для переменных и функций, SCREAMING_SNAKE_CASE для констант.
* Стили: используйте SCSS-модули (`Component.module.scss`) для изоляции; общие переменные — в `variables.scss` и `src/styles`.
* Форматирование: следуйте правилам ESLint; запускайте `lint:fix` перед отправкой. Отступы — 2 пробела, кавычки — одинарные.

## Тестирование

* Автоматические тесты пока не настроены. При добавлении функциональности рекомендуется писать unit-тесты (Vitest + React Testing Library).
* Размещайте тесты рядом с кодом (`*.test.tsx`) или в папке `__tests__/`.
* Покрывайте тестами API-утилиты и поведение страниц (роутинг, пагинация).
* Если добавляете тестовый раннер — добавьте скрипт `npm test`.

## Коммиты и Pull Request'ы

* Коммиты: короткие, в повелительном наклонении (например: `Fix repo card overflow`, `Add pagination hook`). Группируйте связанные изменения, избегайте лишних WIP-коммитов.
* Pull Request: краткое описание, скриншоты/GIF для UI-изменений, шаги для проверки. Указывайте связанные задачи и изменения в конфигурации.
* Перед ревью: выполните `npm run lint` и, если есть, тесты.

## Переменные окружения и доступ к API

* Создайте файл `.env.local` с переменной:

  ```
  VITE_GITHUB_TOKEN=<ваш_personal_access_token>
  ```

  Она необходима для работы с GitHub API. Не коммитьте секреты.
* После изменения `.env` перезапустите `npm run dev`, чтобы Vite подхватил новые значения.
