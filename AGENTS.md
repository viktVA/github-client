# Repository Guidelines

## Структура проекта
- Входная точка Vite + React 19 + TypeScript — `src/main.tsx`; роутинг настроен в `src/config/routesConfig.tsx`, страницы лежат в `src/App/pages`.
- Повторно используемые UI-компоненты в `src/components` (папки PascalCase, SCSS‑модули, BEM‑стили). Общие Sass-переменные в `src/components/variables.scss`, глобальные стили и сбросы — в `src/styles`.
- Запросы к GitHub через axios в `src/api`; доменные типы в `src/custom_types`; утилиты в `src/utils`; статика — `src/assets`; дизайн-референсы — `src/design-reference`.
- Пользуйтесь alias'ами из `tsconfig.json`: `@components/Button`, `@config/routes`, `@api/getRepos` вместо длинных относительных путей.
- В `src/components` лежат компоненты Button, Card, CheckBox, icons, Input, Loader, MultiDropdown, Text.
- В проекте созданы страницы `RepoListPage` - страница списка репозиториев организаций, `RepoPage` - страница отдельного репозитория.


## Сборка, запуск и проверки
- `yarn install` — установка зависимостей (используем Yarn lock).
- `yarn dev` — дев‑сервер Vite с HMR.
- `yarn build` — `tsc -b` + `vite build` (типизация и прод‑сборка).
- `yarn preview` — локальный просмотр прод‑бандла.
- `yarn lint` / `yarn lint:fix` — ESLint (с автоисправлением). Запускайте перед коммитом/PR.
- Тесты пока не подключены; при добавлении первого набора создайте `test` скрипт, чтобы команда была единообразной.

## Стиль кода и именование
- Пишем на TypeScript; порядок импортов: сторонние пакеты, затем alias. Используем точку с запятой и двойные кавычки как в текущих файлах.
- Компоненты/хуки — PascalCase/CamelCase; имя файла совпадает с компонентом (`Button/Button.tsx`).
- Стили: SCSS‑модули для компонентов, глобальные стили — обычный CSS. Классы в духе BEM (`block__element_modifier`), значения берите из `variables.scss`.
- ESLint: базовые правила `@eslint/js`, `typescript-eslint`, React Hooks, React Refresh; Prettier установлен — форматируйте перед коммитом.

## Тестирование
- Стэк тестов ещё не выбран. Рекомендуется класть `*.test.tsx` рядом с компонентами или в `src/__tests__`. Минимум покрывайте: рендер в разных состояниях, обработку ошибок API, переходы роутера. Добавьте скрипт `test` в `package.json`, когда определитесь с раннером.

## Коммиты и Pull Request'ы
- История использует короткие сообщения в настоящем времени (“Fix repository route…”, “Add getRepo…”). Соблюдайте стиль, одна тема — один коммит.
- В PR: короткое описание, что и почему поменяли; какие команды запускавались; скриншоты/гифки для UI; ссылки на задачи; отметки о лимитах GitHub API, если затрагивали.
- Перед ревью убедитесь, что `yarn lint` и `yarn build` проходят; если что-то пропущено — явно укажите причину.

## Работа с GitHub API
- Запросы сейчас без токена, возможны rate limit на активных сессиях. При необходимости добавьте `VITE_GITHUB_TOKEN` в окружение и пробросьте его в axios-заголовки.
- Держите логику API чистой от React и централизуйте эндпоинты в `src/api`, чтобы не плодить дубли.


## Правила кода
- Все компоненты — функциональные, типы через interface Props
- Никаких inline styles — только классы или переменные из `variables.scss`
- Импорт компонентов всегда через `src/components/ui/index.ts`
- Новый компонент = папка с Component.tsx + Component.module.scss + добавить в index.ts, если у него есть свои компонент, который использует только он, в таком случае помещать в папку нового компонента по пути `Component/components/`.
